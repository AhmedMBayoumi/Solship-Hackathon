### 1. SANITY CHECK: The "Noise Floor" vs. The Plan
**Verdict:** This strategy is unlikely to drop NRMSE by ≥1.5pp (e.g., from 60.6% to 59.1%).
**Why:** At 60.65% NRMSE, you are likely hitting the **stochastic entropy limit** of a single meter. Residential "spikes" are often random (manual appliance use) rather than structural. Wavelets are excellent at capturing transient frequencies, but if those frequencies don't repeat with a predictable causal driver (e.g., weather or schedule), they just add 150 more ways to overfit the noise.
**Expected Gain:** **0.3% – 0.7%**. This is a "refinement" move, not a "breakthrough" move.

---

### 2. PRETRAINED MODELS: Use "Wavelet Scattering"
**Recommendation:** Do not use WaveNet or Chronos hidden states. Use the **Wavelet Scattering Transform (WST)** (e.g., via `pywt` or `kymatio`).
- **Why:** WST is a fixed, non-trainable "CNN" that provides shift-invariance and captures multi-scale correlations using the wavelet modulus. It is mathematically "optimal" for extracting features from noisy signals when you don't have enough data to train a deep encoder from scratch.
- **Payoff:** High. It gives you "Deep Learning" style features without the training risk.

### 4. META-LAYER: The Residual LGBM Stacker
**Winner:** **LightGBM Residual Stacker**.
- **Avoid:** DL Meta-layers (MLPs/Transformers). With only two test windows and high noise, a DL layer will memorize the "bias" of your bags on the training set and fail on the test set.
- **Strategy:** Train Path A bags to predict the load. Train a **Meta-LGBM** to predict the **RESIDUAL** (Actual - Bag_Mean).
- **Architecture:** 
    - `Input`: [Mean of 8 Path A Bags] + [Raw Path B Features].
    - `Output`: ΔCorrection.

---

### 5. LEAKAGE TRAP: The "Validation Buffer"
To avoid the stacker thinking Path A is "perfect," you need **Out-of-Fold (OOF)** predictions.
1. **Fold 1:** Train Path A bags on **2024 data**.
2. **Fold 2 (Stacker Train):** Use those bags to predict **Jan–Mar 2025**. This is your "Stacker Training Set."
3. **Fold 3 (Test):** Predict **April 2025**.
*Repeat for the September window using a similar rolling buffer.*

import numpy as np
import pandas as pd
import pywt
import lightgbm as lgb
from sklearn.base import BaseEstimator, TransformerMixin

class WaveletFeatureExtractor(BaseEstimator, TransformerMixin):
    def __init__(self, wavelet='db4', level=4):
        self.wavelet = wavelet
        self.level = level

    def transform(self, X_load_history):
        # X_load_history: (N, 96) - 24 hours of 15-min lags
        feats = []
        for i in range(len(X_load_history)):
            # Multi-resolution analysis
            coeffs = pywt.wavedec(X_load_history[i], self.wavelet, level=self.level)
            # Feature: Energy (sum of squares) and StdDev of each band
            energy = [np.sqrt(np.mean(c**2)) for c in coeffs]
            std = [np.std(c) for c in coeffs]
            feats.append(energy + std)
        return np.array(feats)
