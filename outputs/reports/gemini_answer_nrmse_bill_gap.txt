### 2. The "Decision-Focused" Logic (Approximation)

The paper you cited (Donti et al., 2017) and the **Smart Predict-then-Optimize (SPO)** framework (Elmachtoub & Grigas, 2022) suggest that the "loss" should be the **Regret**: the bill difference between using the forecast vs. using the oracle.

Since you can't retrain, you can mimic this by **warping the forecast space**. The LP dispatcher is "risk-averse" regarding high-tariff imports. To minimize the bill, you want a forecast that acts as a **soft upper bound during high-price periods**.

**The Trick:** Map your point forecast to a higher quantile (e.g., $q=0.65$) instead of the mean, but **only during F1/F2 tariff bands**.

---

### 3. Is it Impossible to improve both? (The Pareto Frontier)

It is **not impossible**, but you are currently moving along a "Bias-Variance" trade-off. 
- **Raw Blend:** High variance (noise), Low bias (peaks are preserved).
- **Smoothed:** Low variance (clean), High bias (peaks are rounded off).

The "Sweet Spot" exists where you **denoise the base load** (improving NRMSE) but **preserve or exaggerate the spikes** (improving the Bill). In image processing, this is why we use **Bilateral Filters** instead of Gaussian blurs—we want to smooth the "texture" (noise) but keep the "edges" (peaks).

---

### 4. Concrete Code Suggestion: The "Tariff-Aware Sharpener"

This technique smooths the low-stakes periods (F3) to harvest NRMSE gains while keeping F1/F2 periods raw (or slightly boosted) to protect the bill.

```python
import pandas as pd
import numpy as np

def apply_decision_focused_postproc(df_preds, df_features, boost_factor=1.05):
    """
    df_preds: columns [timestamp, load_pred]
    df_features: columns [timestamp, buy_price, tariff_band]
    """
    # 1. Join with tariff data
    df = df_preds.merge(df_features[['timestamp', 'buy_price', 'tariff_band']], on='timestamp')
    
    # 2. Create a smoothed version (for NRMSE)
    df['load_smooth'] = df['load_pred'].rolling(window=3, center=True, min_periods=1).mean()
    
    # 3. Hybrid Logic: 
    # Use smoothed for F3 (off-peak) - helps NRMSE, doesn't hurt bill much.
    # Use RAW for F1/F2 (peak) - preserves timing for the LP.
    # Optionally boost F1 by 5% to force the LP to be conservative.
    
    is_f1 = (df['tariff_band'] == 0) # F1 is usually 0 in your label encoding
    is_f3 = (df['tariff_band'] == 2) # F3 is usually 2
    
    df['final_pred'] = df['load_smooth'] # Default to smooth
    
    # Restore raw peaks for high-tariff periods
    df.loc[~is_f3, 'final_pred'] = df.loc[~is_f3, 'load_pred']
    
    # Apply "Decision-focused" boost to F1 peaks
    df.loc[is_f1, 'final_pred'] *= boost_factor
    
    return df[['timestamp', 'final_pred']].rename(columns={'final_pred': 'load_pred'})
```

### 5. Recommended "Hail Mary" for the last hour:
Run a **Bilateral Filter** in 1D. It is the most rigorous way to say "smooth the small stuff, leave the big stuff alone."

```python
from scipy.ndimage import gaussian_filter1d

def bilateral_filter_1d(x, sigma_s=1.0, sigma_r=0.5):
    """
    sigma_s: Time domain smoothing (like MA window)
    sigma_r: Value domain smoothing (how much difference constitutes a 'peak')
    """
    out = np.zeros_like(x)
    for i in range(len(x)):
        weights = np.exp(-0.5 * ((np.arange(len(x)) - i)**2 / sigma_s**2))
        # This is the magic: reduce weight if the value is very different from x[i]
        weights *= np.exp(-0.5 * ((x - x[i])**2 / sigma_r**2))
        out[i] = np.sum(weights * x) / np.sum(weights)
    return out
```
