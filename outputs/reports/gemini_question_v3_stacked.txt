Problem: 15-min residential load forecasting (Sondrio, Italy, single household).
Test windows: April 2025 + September 2025.
Best so far: 60.65% NRMSE (LightGBM walkforward + online retraining, 12 bags).
Floor: CV(load)=1.15, lag-1 ACF=0.80 → naive lag-1 ≈ 69%, so 60.65% IS already

  - Two-stage spike classifier (61.45%, worse)

  - Chronos zero-shot (76%, much worse)
  - 60-trial Optuna (61.08%)
  - v6 Italian behaviour features (60.76%)
  - v7 causal wavelet decomposition (60.75%)
  - v8 PCA + autoencoder + FFT representations (61.00%)

THE NEW PROPOSAL (please sanity-check + refine)
================================================
The user wants to combine wavelet-domain feature extraction with stacked
ensembling. Two parallel paths feeding a final DL meta-layer:

PATH A  (wavelet path)
  step A1: transform load history into wavelet domain (multi-resolution)
  step A2: extract latent features using EITHER
           - a pretrained model that works on wavelet/spectral data, OR
           - an encoder we train from scratch on the wavelet bands
  step A3: train 8 diverse LGBM bags on those latent features
  step A4: collect the 8 bag predictions on the test windows

PATH B  (raw path)
  the 8 bag predictions from path A   +
  the original tabular features (lags, calendar, weather, PHANN clear-sky,
  Italian behaviour features, etc.)
  → flow straight into a final DL meta-layer

FINAL DL META-LAYER
  takes [8 bag preds]  ⊕  [path-B raw tabular features]
  → outputs the final 15-min load forecast
  trained walkforward on a held-out validation window so it learns when to
  trust the wavelet path vs when to override with raw signals.
